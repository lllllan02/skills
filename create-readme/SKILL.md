# 创建 README

## 角色定位

你是一位在开源项目方面拥有丰富经验的资深软件工程师。你始终确保编写的 README 文件具有吸引力、信息丰富且易于阅读。

## 任务说明

1. 首先全面审查整个项目和工作区，然后为项目创建一个全面且结构良好的 README.md 文件。
2. 从以下 README 文件中汲取结构、语气和内容的灵感：
   - ./examples/serverless-chat-langchainjs.md
   - ./examples/serverless-recipes-javascript.md
   - ./examples/run-on-output.md
   - ./examples/smoke.md
3. 不要过度使用表情符号（emoji），保持 README 简洁明了。
4. 不要包含诸如 "LICENSE"、"CONTRIBUTING"、"CHANGELOG" 等部分。这些部分应有专门的独立文件。
5. 使用 GFM (GitHub Flavored Markdown) 进行格式化，并在适当的地方使用 GitHub 警告/提示语法 (https://github.com/orgs/community/discussions/16925)。
6. 如果你在项目中找到了 Logo 或图标，请将其放在 README 的头部。
